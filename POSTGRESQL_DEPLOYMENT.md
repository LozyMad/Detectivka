# 🐘 Деплой с PostgreSQL

## 📋 Подготовка к деплою

### 1. Локальное тестирование (опционально)

Если у вас установлен PostgreSQL локально:

```bash
# Создайте тестовую базу данных
createdb detectivka_test

# Запустите тест подключения
node test-postgresql.js
```

### 2. Миграция данных из SQLite (если нужно)

Если у вас есть данные в SQLite, которые нужно перенести:

```bash
# Убедитесь что PostgreSQL запущен локально
# Создайте базу данных: createdb detectivka

# Запустите миграцию
node migrate-to-postgresql.js
```

## 🚀 Деплой на хостинг (Timeweb и др.)

### 1. Создание PostgreSQL базы данных

1. В панели хостинга создайте базу PostgreSQL (или используйте существующую)
2. Запомните хост, порт, имя БД, логин и пароль

### 2. Настройка переменных окружения

В панели хостинга (переменные окружения / .env) добавьте:

```env
DB_TYPE=postgresql
NODE_ENV=production
JWT_SECRET=your-super-secret-jwt-key-change-this
PORT=3000
DATABASE_URL=postgresql://username:password@host:port/database
```

`DATABASE_URL` — строка подключения к вашей PostgreSQL (её даёт панель хостинга при создании БД).

### 3. Деплой кода

```bash
git add .
git commit -m "Your changes"
git push origin main
```

При настроенном автодеплое из Git изменения подхватятся на сервере автоматически.

### 4. Проверка деплоя

1. Откройте логи приложения на хостинге
2. Убедитесь, что видите сообщения:
   - `Initializing PostgreSQL database...`
   - `PostgreSQL database initialized successfully`
   - `Server is running on port 3000`

## 🔧 Переключение между SQLite и PostgreSQL

Проект поддерживает оба типа баз данных. Для переключения:

### Локально (SQLite):
```env
DB_TYPE=sqlite
# или просто не указывайте DB_TYPE
```

### На продакшене (PostgreSQL):
```env
DB_TYPE=postgresql
DATABASE_URL=postgresql://username:password@host:port/database
```

## 📊 Преимущества PostgreSQL

✅ **Надёжность**: данные не пропадают при деплоях  
✅ **Масштабируемость**: поддержка множественных подключений  
✅ **Универсальность**: работает на любом хостинге  
✅ **Бэкапы**: настраиваются на стороне хостинга  
✅ **Производительность**: лучше для продакшена  

## 🛠️ Устранение проблем

### Ошибка подключения к базе данных
```
Error: connect ECONNREFUSED
```
**Решение**: убедитесь, что PostgreSQL создана на хостинге и `DATABASE_URL` задана верно.

### Ошибки миграции
```
Error: relation "users" does not exist
```
**Решение**: база инициализируется при первом запуске. Дождитесь завершения деплоя и перезапуска.

### Проблемы с синтаксисом SQL
**Решение**: все запросы адаптированы под PostgreSQL. При ошибках проверьте, что `DB_TYPE=postgresql`.

## 📞 Поддержка

Если возникают проблемы:
1. Проверьте логи приложения на хостинге
2. Убедитесь, что все переменные окружения установлены
3. Проверьте, что база PostgreSQL создана и доступна

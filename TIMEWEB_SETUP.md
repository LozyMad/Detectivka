# Запуск на Timeweb (облачный сервер)

Если в панели сервер «работает», но сайт по IP не открывается (`ERR_CONNECTION_REFUSED`), обычно одно из трёх:

1. **Node.js приложение не запущено** или упало  
2. **Приложение слушает только localhost** — исправлено: теперь слушаем `0.0.0.0`  
3. **Закрыт порт файрволом** (на сервере или в панели Timeweb)

---

## 1. Подключись к серверу по SSH

Из панели Timeweb открой консоль или подключись по SSH с своего ПК.

---

## 2. Убедись, что приложение вообще запущено

После деплоя кода приложение нужно **запустить**. Один раз запущенное в консоли `node backend/server.js` умрёт при закрытии сессии.

**Вариант А — PM2 (удобно для продакшена):**

```bash
# Установка PM2 глобально
sudo npm install -g pm2

# Перейди в папку проекта (путь может быть другой, например /var/www/detectivka или ~/Detectivka)
cd /путь/к/проекту

# Запуск (из корня репозитория, где есть backend/ и package.json)
pm2 start backend/server.js --name detectivka

# Автозапуск после перезагрузки сервера
pm2 startup
pm2 save

# Логи, если что-то не так
pm2 logs detectivka
```

**Вариант Б — systemd** (если не хочешь PM2): создай юнит и запускай приложение как сервис.

Проверка: на сервере выполни `curl http://127.0.0.1:3000` (или тот порт, что в `PORT`). Если отвечает — приложение работает локально.

---

## 3. Открой порт в файрволле на сервере

Если используешь **ufw**:

```bash
# Разрешить порт приложения (например 3000)
sudo ufw allow 3000/tcp

# Или 80, если поднял nginx и проксируешь на 3000
sudo ufw allow 80/tcp

sudo ufw reload
sudo ufw status
```

Если порт не открыт — снаружи будет именно `ERR_CONNECTION_REFUSED` (или таймаут).

---

## 4. Проверь файрвол / группы безопасности в Timeweb

В панели Timeweb для этого сервера посмотри:

- **Сети / файрвол / группы безопасности** — есть ли правило, разрешающее входящий трафик на порт **3000** (или 80, если заходишь по 80).
- Если входящие на этот порт запрещены — добавь правило: протокол TCP, порт 3000 (или 80).

---

## 5. Как заходить на сайт

- Если открыл порт **3000**: в браузере вводи `http://5.42.113.79:3000`  
- Если поставил nginx на **80** и проксируешь на 3000: `http://5.42.113.79`

---

## 6. Переменные окружения на сервере

Убедись, что на сервере заданы нужные переменные (в `.env` в папке backend или в PM2/systemd):

- `PORT=3000` (или 80, если без nginx)
- `NODE_ENV=production`
- `JWT_SECRET=...`
- Для PostgreSQL: `DB_TYPE=postgresql`, `DATABASE_URL=...`

---

## Краткий чеклист

| Шаг | Действие |
|-----|----------|
| 1 | Код на сервере (git clone / деплой) |
| 2 | `npm install` в корне и при необходимости в `backend/` |
| 3 | Запуск приложения через **PM2** или systemd (не просто в консоли) |
| 4 | Приложение слушает **0.0.0.0** (уже сделано в коде) |
| 5 | Порт 3000 (или 80) открыт в **ufw** на сервере |
| 6 | В панели Timeweb входящий трафик на этот порт **разрешён** |
| 7 | Заход по `http://IP:3000` или `http://IP` (если порт 80) |

После этого «сервер работает» в панели будет означать и реально работающий сайт по IP.

# Автодеплой с нуля: полная настройка цепочки

Пошаговая инструкция: удалить старое и заново настроить интеграцию GitHub → сервер.

---

## Шаг 1. Удалить старое на GitHub

1. Открой репозиторий **Detectivka** на GitHub.
2. **Settings** → **Secrets and variables** → **Actions**.
3. Найди секрет **DEPLOY_SECRET** → нажми на него → **Remove** (удали).
4. Если в **Settings** → **Webhooks** есть вебхук с URL вида `.../api/deploy` — удали его (кнопка с тремя точками → Delete).  
   *(Мы используем GitHub Actions, а не Webhooks — вебхук можно не создавать.)*

---

## Шаг 2. Сгенерировать новый секрет

Нужна длинная случайная строка. Варианты:

**Вариант А — на своём компьютере (PowerShell):**
```powershell
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])
```
Скопируй вывод (одна строка без пробелов) — это твой **новый секрет**.

**Вариант Б — на сервере в консоли:**
```bash
openssl rand -hex 24
```
Скопируй вывод — это твой **новый секрет**.

**Вариант В — придумать вручную:** например строка из 30+ символов букв и цифр без пробелов.

Сохрани этот секрет в блокнот — он понадобится в двух местах (сервер и GitHub).

---

## Шаг 3. Настроить секрет на сервере

1. Подключись к серверу по SSH (консоль Timeweb).
2. Перейди в папку проекта:
   ```bash
   cd /root/Detectivka
   ```
3. Открой или создай файл с переменными:
   ```bash
   nano backend/.env
   ```
4. Убедись, что есть строка (подставь **свой** сгенерированный секрет):
   ```env
   DEPLOY_SECRET=твой_новый_секрет_без_пробелов
   ```
   Если такой строки нет — добавь её. Если была старая — замени на новую.  
   Другие переменные (NODE_ENV, PORT, JWT_SECRET и т.д.) оставь как есть.
5. Сохрани: **Ctrl+O**, Enter, выход: **Ctrl+X**.
6. Перезапусти приложение:
   ```bash
   pm2 restart detectivka
   ```
7. Проверка (подставь свой секрет):
   ```bash
   curl -X POST 'http://127.0.0.1:3000/api/deploy' -H 'X-Deploy-Secret: твой_новый_секрет'
   ```
   Должен вернуться ответ с **200** и телом вида `{"ok":true,"log":"..."}`. Если **403** — секрет на сервере не совпадает с тем, что в заголовке.

---

## Шаг 4. Добавить секрет в GitHub

1. Репозиторий на GitHub → **Settings** → **Secrets and variables** → **Actions**.
2. Нажми **New repository secret**.
3. **Name:** `DEPLOY_SECRET` (точно так, заглавными).
4. **Value:** вставь **тот же самый** секрет, что записан в `backend/.env` на сервере (без пробелов и кавычек).
5. Нажми **Add secret**.

---

## Шаг 5. Проверить цепочку

1. Сделай маленькое изменение в проекте (например, в админке текст «Комнаты!» → «Комнаты» или наоборот), закоммить и запушь в `main`.
2. Открой **Actions** → workflow **Deploy to Timeweb** → последний запуск.
3. Должен быть **успех** (зелёная галочка), в логе шага «Call deploy webhook on server» — **Response code: 200**.
4. Через 20–30 секунд открой сайт и обнови страницу с **Ctrl+F5** — изменения должны появиться.

---

## Что есть в проекте (ничего удалять не нужно)

- **Сервер:** в `backend/server.js` уже есть эндпоинт `POST /api/deploy`. Он читает `DEPLOY_SECRET` из переменных окружения (через `backend/.env`).
- **GitHub:** в `.github/workflows/deploy.yml` при каждом `push` в `main` отправляется запрос на `http://5.42.113.79:3000/api/deploy` с заголовком `X-Deploy-Secret: ${{ secrets.DEPLOY_SECRET }}`.

Тебе нужно только: **один и тот же секрет** в `backend/.env` на сервере и в **Actions → DEPLOY_SECRET** на GitHub. После этого цепочка будет работать.

---

## Если IP или порт сервера изменились

Отредактируй в репозитории файл **`.github/workflows/deploy.yml`**: в строке с `curl` замени URL на актуальный (например `http://НОВЫЙ_IP:3000/api/deploy` или с доменом). Затем закоммить и запушь.

# Чеклист: почему сайт не обновляется после коммита

## 1. Проверить на GitHub (Actions)

1. Открой репозиторий → вкладка **Actions**.
2. Выбери workflow **«Deploy to Timeweb»**.
3. Открой **последний запуск** (после твоего последнего пуша).
4. Нажми на job **deploy** → шаг **«Call deploy webhook on server»**.
5. В логе найди строки:
   - **Response code:** ???
   - **Response body:** ???

| Код | Что значит | Что делать |
|-----|------------|------------|
| **200** | Запрос дошёл, деплой должен был выполниться | Перейти к п. 2 (проверка на сервере). |
| **501** | На сервере не задан `DEPLOY_SECRET` | На сервере создать/проверить `backend/.env` и перезапустить приложение (п. 3). |
| **403** | Неверный секрет | Секрет в GitHub (Secrets → DEPLOY_SECRET) должен **совпадать** с `DEPLOY_SECRET` в `backend/.env` на сервере. |
| **000** или ошибка curl | Запрос не дошёл до сервера | Открыть порт 3000 для входящих с интернета (файрвол на сервере или в панели Timeweb). |

---

## 2. Проверить на сервере (если в Actions был 200)

Зайди по SSH на сервер и выполни:

```bash
cd /root/Detectivka   # или твой путь к проекту
git log -1 --oneline
```

- Если там **не** последний коммит с GitHub (например, не `ac3a335` или новее) — на сервере не выполнился `git pull`. Проверь логи: `pm2 logs detectivka` — возможно, ошибка при `git pull` или `pm2 restart`.
- Если коммит совпадает — код подтянулся. Обнови страницу сайта с **Ctrl+F5** (сброс кэша браузера).

---

## 3. Настроить DEPLOY_SECRET на сервере (если в Actions был 501)

На сервере:

```bash
cd /root/Detectivka
```

Проверить, есть ли файл и переменная:

```bash
cat backend/.env | grep DEPLOY_SECRET
```

Если файла нет или переменной нет:

```bash
echo "DEPLOY_SECRET=твой_секрет_как_в_github" >> backend/.env
```

(подставь тот же секрет, что в GitHub → Settings → Secrets and variables → Actions → **DEPLOY_SECRET**.)

Обязательно перезапустить приложение (чтобы подхватить `.env`):

```bash
pm2 restart detectivka
```

Проверка с сервера:

```bash
curl -s -o /dev/null -w "%{http_code}" -X POST "http://127.0.0.1:3000/api/deploy" -H "X-Deploy-Secret: твой_секрет"
```

Должно вернуться **200**.

---

## 4. Проверить на GitHub (Secrets)

1. Репозиторий → **Settings** → **Secrets and variables** → **Actions**.
2. Должен быть секрет **DEPLOY_SECRET**.
3. Его значение должно **совпадать** с `DEPLOY_SECRET` в `backend/.env` на сервере (без лишних пробелов и кавычек).

---

## 5. Итог

- **Сайт не обновляется** → открой Actions и посмотри **Response code** в последнем запуске.
- **501** → на сервере нет или не читается `DEPLOY_SECRET` → п. 3.
- **403** → не совпадает секрет → п. 4 и п. 3.
- **200** → деплой вызван → п. 2 (проверить `git log` и кэш браузера).

# Автоподтягивание кода на сервер (вебхук)

При пуше в GitHub сервер может автоматически выполнять `git pull` и перезапуск приложения.

## 1. На сервере

1. Задай переменную окружения (придумай длинный случайный секрет):
   ```bash
   # В .env в папке backend или в панели Timeweb
   DEPLOY_SECRET=твой_секретный_ключ_например_abc123xyz
   ```

2. Перезапусти приложение, чтобы подхватить переменную:
   ```bash
   pm2 restart detectivka
   ```

## 2. В GitHub

1. Открой репозиторий → **Settings** → **Webhooks** → **Add webhook**.

2. Укажи:
   - **Payload URL:** `https://ТВОЙ_ДОМЕН_ИЛИ_IP:3000/api/deploy`  
     (например: `https://5.42.113.79:3000/api/deploy` или `https://detectivka.example.com/api/deploy`)
   - **Content type:** `application/json`
   - **Secret:** тот же самый ключ, что в `DEPLOY_SECRET` на сервере (сервер проверяет подпись GitHub этим секретом)
   - **Which events:** Just the push event

3. Сохрани вебхук (Add webhook).

При каждом пуше в репозиторий GitHub отправит запрос на этот URL; сервер проверит подпись и выполнит `git pull` и `pm2 restart detectivka`.

## 3. Ручной вызов (без GitHub)

Можно дергать деплой вручную или из скрипта, передав секрет в заголовке:

- `X-Deploy-Secret: твой_секретный_ключ`
- или `Authorization: Bearer твой_секретный_ключ`

## 4. Проверка вручную (curl)

```bash
curl -X POST https://ТВОЙ_IP:3000/api/deploy \
  -H "X-Deploy-Secret: твой_секретный_ключ"
```

В ответе должно быть `{"ok":true,"log":"..."}`.

## 5. Важно

- Секрет храни только в переменных окружения на сервере (и в GitHub Secrets, если используешь Actions). Не коммить его в репозиторий.
- Порт 3000 должен быть доступен с интернета (или используй домен с проксированием на 3000).
- После `git pull` выполняется `pm2 restart detectivka` — имя процесса должно совпадать с тем, что у тебя в PM2.
